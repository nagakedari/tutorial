How Geth Works?

Let's us discuss how the blockchain nodes communicate each other. Lets say you download Geth (command line tool) and running your local. When you open Geth it does two things.

  1. Connects to other nodes in the blockchain and downloads the blocks on to your local machine
  2. Opens a Javascript JSON RPC for you to interact with geth node and hence the blockchain

1. Connecting and Syncing: In this step, Geth connects to other nodes and try to download the blocks on to the local machine by syncing with the network. There are 3 types of syncing.

  • Full Sync - In this mode, geth downloads all the blocks (Block Headers + Block Bodies) from the begining and re process all the transactions. Hence it requires lot of disk space and takes for ever
  • Fast Sync - In this mode, geth downloads all the blocks and process last 1024 transactions. Hence it requires less space when compared to Full sync and takes hours for completion
  • Light Sync - In this mode, geth downloads only the the header portion of a block and process current snapshot of transactions. It requires very less space and takes only few minutes to complete. But this is new way of syncing and its still in experimental phase, hence no gurentee that it works all times.

2. Javascript JSON RPC: In this step, Geth opens Javascript RPC so that you can issue commands to blockchain. There are two ways you can interact with Javascript RPC.

  • IPC (Inter Process Communication) : Interact using file or pipe
  • Http RPC : This enables a browser to interact with Javascript RPC. This is where web3.js will come in to picture.
