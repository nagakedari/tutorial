What is Blockchain?

Blockchain is a public ledger of all the transactions ever occured all the way from the initial transaction. As the name says, its a chain of blocks that holds the data. Once the data is written on block it is highgly impossible to change/tamper the data. Let's understand a block and see why it is impossible to change the data. A block contains data, hash of the block and the hash of a previous block.

The data that a block contains depends on the type of blockchain. In ehtereum, the data would be a smart contract. In bitcoin, the data will be transactional data between sender and reciever. The hash in the block is used to identify the block. Whenever a block gets created, a new hash will be calculated. Similarly if we make some changes to the existing block, the hash value will be re-calculated. The third content that a block has is the hash code for previous block, which makes it a chain. The very first block which is also called as genesis block will have the previous block as empty.

Blockchain constantly grows as new blocks are added. The transactions that are added on to the network are being processed by the computers connected to the nework. These computers are referred as Nodes. These nodes are scattered through out the world, hence its a decentralized technology. Any new block added to the chain turns out in to chronological order. Anything that happens on the network happens as a whole.

Let's say hypothetically, if we tampered the second block (hashcode: 1RT5) in the above diagram, The hash of that block changes and subsequent blocks becomes invalid because they no longer store the previous block hash. So changing a single block makes all the following blocks in valid. But hash code alone is not sufficient to make the blockchain secure. Because we can tamper a block and recalculate all the subsequent blocks to make the block chain valid again. To prevent this, blockchain has something called Proof-Of-Work. It's a mechanism that slows down the creation of new blocks. This makes very hard to tamper the data, because if you tamper one block you will have to re-calculate the proof-of-work for all the subsequent blocks. So the security of blockchain comes from both the hash code and the mechanism of proof-of-work.

Since blockchain is decentralized, any new block created should be verified by all nodes in the network. All nodes in the network will have exactly the same blockchain data. Whenever a new block is created, it gets copied to each node. Each node then verifies it and approves it consenting with all other nodes. Tampered nodes will be rejected by the network. So to sum up, If someone wants to tamper a block, they have to recalculate the hash codes and proof-of-work for all the following blocks and take control of more than 50% of the nodes which is almost impossible.

The blockchain is designed for longevity and durability. Its not maintained by a single entity. All the nodes that are participated in a network are part of one community. Hence it has no single point of failure.
