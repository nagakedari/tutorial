Ethereum Development Tools

Before we start learning how to code in ethereum blockchain, let's get familiarize with the tools that we will be using for development. The following are the tools and plugins that we will use in this tutorial.

  • Geth (or) Go-Ethereum
  • Remix
  • Solidity
  • Web3.js

Geth or Go-Ethereum: Go-Ethereum often refered as Geth is a blockchain node which is running on client side. It's an Ethereum CLI and is the entry point to Ethereum network. Geth can run in full node mode (default behaviour), archive node (all historical data) or in a light node mode.It can be used by other processes as a gateway into the Ethereum network via JSON RPC endpoints exposed on top of HTTP, WebSocket and/or IPC transports.

Remix: Remix is an IDE running on Cloud. You don't have to install it and can simply access it from Remix.ethereum.org. You can use remix to compile solidity code, run and debug it. It has in-memory blockchain, which makes very easy to test your solidity programs/contracts.

Solidity: Solidity is a contract-oriented, high-level language for implementing smart contracts. It was influenced by C++, Python and JavaScript. A solidity program gets compiled in to byte code using EVM (Ethereum Virtual Machine) similar to other programming languages, say for example JAVA.

Web3.js: This is the Ethereum compatible JavaScript API which implements the Generic JSON RPC spec. It's also available on npm as a node module. Web applications which require blockchain interaction can use this javascript API to communicate with blockchain.
