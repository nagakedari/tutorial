Getting Started With Solidity

Solidity is a high level programming language which is used to build smart contracts. A contract in solidity looks like this. 

{{
     
     'pragma solidity ^0.4.0;

contract SampleContract{
    uint value;
    
    function SampleContract(){
        value = 10;
    }
    
    function getValue()constant returns (uint){
        return value;
    }
    function setValue(uint newValue){
        value = newValue;
    }
}'
     }}

The first statement pragma solidity ^0.4.0 tells the version of solidity that will be used to compile. The contract is similar to a class in java/c++, which has variables/properties and functions that operates on those variables. value is a contract variable of type unassigned integer. The function SampleContract() is a constructor of the contract which gets executed first when the contract runs. The function setValue sets the unassigned integer variable to some value. The getValue function returns the value of the contract variable. The getter and setters are more like the getters and setters in other languages (ex: Java). The extension of solidity contract file is .sol

We can write, compile and run the solidity programs in Remix which is running on cloud (Remix). In remix, careate a new file, copy and paste the above code and go to Run tab. You will have Environment, Account, Gas Limit and Value configurable parameters used while running a contract. Environment will have 3 options, JavaScript VM will execute the contract using in memory network. That means it will have the network and blockchain everything avaialable in memory. This is something remix provides and we can use this option for testing the contracts we build in remix. Injected Web3 option will inject the web3 object provided my Mist or Metamask browser plug in and interacts with the network provided by these plugins. Web3 Provider option will try to connect to a node running on a network. For example in earlier sections (Refer Creating private network section), we created a private netwrok and using web3 provider option, we can connect to that network, deploy and run the contract on that private network. The easiest options is to use in-memory network which is Javascript VM. Account is the one associated to your wallet. An account will hold the ethers. Gas Limit is the maximum gas that a transaction sends and value is used to send some ether to the contract.

To run the above contract, go to Run tab, select the environement as Javascript VM, select a test account that remix proides, use the default gas limit and click on Create button. Create button will deploy the contract to in-memory network. You can see get and set methods as buttons. If you click on getValue button, it should display the value as 10 which we set in the constructor. You can also set the value by passing some value. Now if you click getValue you will get the updated value.

    

Solidity program gets compiled into byte code using EVM (Ethereum Virtual Machine). Each instruction costs some gas. If the gas reaches the passed in Gas limit, the contract stops running automatically. The set function or any write instructions costs gas. Since get function is only reading the data from the block chain so it doesn't need the gas. That's why if you noticed the getValue function, we have a keyword constant.
